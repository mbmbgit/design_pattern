GitHub Pagesに認証機能を追加する方法 (Firebase Authentication編)GitHub Pagesは静的なホスティングサービスのため、サーバーサイドのコード（データベース接続やセッション管理など）を実行できません。そのため、Googleが提供する Firebase Authentication を利用して、クライアントサイド（JavaScript）で認証を行います。手順 1: Firebase プロジェクトの作成Firebase コンソール にアクセスし、Googleアカウントでログインします。「プロジェクトを追加」 をクリックし、適当な名前（例: my-github-page-auth）を入力してプロジェクトを作成します。Google アナリティクスの設定は、今回はオフでも構いません。手順 2: アプリの登録と設定情報の取得プロジェクトの概要ページで、ウェブアイコン（</>） をクリックします。アプリのニックネームを入力（例: My Page）し、「このアプリの Firebase Hosting も設定します」のチェックは外したまま、「アプリを登録」をクリックします。「Firebase SDK の追加」 画面に表示される const firebaseConfig = { ... }; のコードブロックをコピーして控えておきます。（後でHTMLファイルに貼り付けます）手順 3: 認証方法（Google / メール）の有効化左メニューの 「構築」 -> 「Authentication」 をクリックします。「始める」 ボタンをクリックします。「ログイン方法」 タブで、以下のプロバイダを有効にします。メール / パスワード: 「有効にする」をオンにして保存。Google: 「有効にする」をオンにし、プロジェクトのサポートメールアドレスを選択して保存。手順 4: GitHub Pages ドメインの許可Firebaseはセキュリティのため、許可されたドメイン以外からの認証リクエストを拒否します。Authentication 画面の 「設定」 タブを開きます。「承認済みドメイン」 までスクロールします。「ドメインを追加」 をクリックし、あなたのGitHub Pagesのドメイン（例: yourname.github.io）を入力して追加します。※ ローカルテスト用に localhost は最初から追加されています。手順 5: コードの実装作成したHTMLファイル（index.htmlなど）に、手順2で取得した firebaseConfig を埋め込み、GitHubにプッシュします。
